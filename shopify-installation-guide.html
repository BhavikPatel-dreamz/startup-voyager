üîß Method 1: Comprehensive Tracking Script Installation

Via Shopify Admin (Recommended)

  1. Go to your Shopify Admin ‚Üí Online Store ‚Üí Themes
  2. Click "Actions" ‚Üí "Edit code" on your active theme
  3. In the left sidebar, find "Layout" ‚Üí "theme.liquid"
  4. Find the </head> tag
  5. Add the tracking script just before the closing </head> tag
<!-- Comprehensive Tracking Script --> <script data-store-id="YOUR_CLIENT_ID_HERE" data-api-key="YOUR_API_KEY_HERE" data-endpoint="https://webhook.site/9752a9dd-5a2d-4f90-87be-c9cdd5102aeb" data-debug="true" data-auto-track="true"> </script> <script src="https://your-domain.com/trackingscript.js"></script>

Via Shopify App

  1. Create a Shopify app in your Partner account
  2. Use the ScriptTag API to inject the tracking script
  3. This method allows for easier updates and management

Via Google Tag Manager

  1. Install Google Tag Manager in your Shopify store
  2. Create a custom HTML tag
  3. Paste the tracking script with your configuration
  4. Trigger on all pages

Comprehensive Script Configuration

<script data-store-id="store_abc123" <!-- Required: Unique store identifier --> data-api-key="api_key_xyz789" <!-- Required: API key for authentication --> data-endpoint="https://your-api.com" <!-- Optional: Custom endpoint URL --> data-debug="true" <!-- Optional: Enable debug logging --> data-auto-track="true" <!-- Optional: Enable automatic tracking --> data-session-timeout="1800000" <!-- Optional: Session timeout in ms (30min) --> > </script>

Analytics Interceptor Configuration

<script data-shopify-interceptor data-store-id="store_abc123" <!-- Required: Unique store identifier --> data-api-key="api_key_xyz789" <!-- Required: API key for authentication --> data-endpoint="https://your-api.com" <!-- Optional: Custom endpoint URL --> data-debug="true" <!-- Optional: Enable debug logging --> data-events='{"Viewed Product": true, "Added Product": true}' <!-- Optional: Custom events --> > </script>
Configuration Parameters:
  • data-store-id: Unique identifier for your store (required)
  • data-api-key: API key for authentication (required)
  • data-endpoint: Custom analytics endpoint (optional)
  • data-debug: Enable console logging (optional, default: false)
  • data-auto-track: Enable automatic tracking (comprehensive script only)
  • data-session-timeout: Session timeout in milliseconds (comprehensive script only)
  • data-events: Custom event configuration (interceptor only)

üìä Step 5: Manual Event Tracking

You can also manually trigger events using the global ea() function:

// Manual event tracking examples ea('trackEvent', 'custom_action', { action: 'button_click', location: 'hero_section' }); ea('trackProductView', { id: 'product_123', name: 'Awesome Product', price: 29.99, category: 'Electronics' }); ea('trackAddToCart', { id: 'product_123', name: 'Awesome Product', price: 29.99, quantity: 2 }); ea('trackSearch', 'wireless headphones', 15);

üîß Step 6: Advanced Customization

Custom Event Listeners

Add custom tracking for specific elements:

// Custom tracking for specific buttons document.addEventListener('click', function(e) { if (e.target.matches('.custom-track-button')) { ea('trackEvent', 'custom_button_click', { button_id: e.target.id, button_text: e.target.textContent }); } }); // Custom tracking for form submissions document.addEventListener('submit', function(e) { if (e.target.matches('.contact-form')) { ea('trackEvent', 'contact_form_submitted', { form_id: e.target.id }); } });

üéØ When to Use Each Method

Use Comprehensive Script When:

  • You need maximum event coverage and custom tracking
  • You want to track custom business events
  • You need detailed product and user behavior data
  • You want to track interactions that Shopify doesn't track
  • You need granular control over what gets tracked
  • You're building a comprehensive analytics platform

Use Analytics Interceptor When:

  • You want minimal performance impact
  • You only need Shopify's native events
  • You want maximum reliability
  • You're concerned about script conflicts
  • You want lightweight implementation
  • You're building a simple analytics solution

Use Hybrid Approach When:

  • You want maximum tracking coverage
  • You need redundancy for critical events
  • You're building an enterprise analytics solution
  • You want to ensure no events are missed
  • You're tracking high-value transactions
  • You need comprehensive data for business decisions
